# LangChain Core RAG Chatbot

A RAG (Retrieval-Augmented Generation) pipeline that answers questions about the [LangChain Core](https://github.com/langchain-ai/langchain) source code.

## How it works

1. Parses `langchain-core` Python source using AST-based splitting
2. Embeds chunks with OpenAI `text-embedding-3-small` (dense) and BM25 (sparse)
3. Stores vectors in Pinecone with hybrid search (dotproduct metric)
4. Answers questions using Claude (`claude-sonnet-4-6`) strictly from retrieved context

## Tech Stack

| Component | Tool |
|---|---|
| Parsing | LangChain `GenericLoader` + `LanguageParser` |
| Chunking | LangChain `RecursiveCharacterTextSplitter` |
| Dense embedding | OpenAI `text-embedding-3-small` |
| Sparse embedding | `BM25Encoder` (pinecone-text) |
| Vector store | Pinecone serverless |
| LLM | Anthropic Claude (`claude-sonnet-4-6`) |
| Evaluation | RAGAS |
| UI | Streamlit |

## Setup

### 1. Clone this repo and install dependencies

```bash
git clone <your-repo-url>
cd rag
uv sync
```

### 2. Clone LangChain source (required for indexing)

```bash
git clone https://github.com/langchain-ai/langchain.git
```

### 3. Add credentials

Create a `.env` file in the project root:

```
PINECONE_API_KEY=your_pinecone_key
OPENAI_API_KEY=your_openai_key
ANTHROPIC_API_KEY=your_anthropic_key
```

### 4. Run the indexing pipeline (once)

```bash
.venv/bin/jupyter notebook notebooks/rag_pipeline.ipynb
```

Run all cells top to bottom. This will:
- Parse and chunk `langchain-core` source
- Fit and save BM25 to `data/bm25_params.json`
- Embed with OpenAI and upsert to Pinecone

> **Note:** Module 5 calls the OpenAI embedding API and incurs a small cost (~$0.01 for this corpus).

### 5. Launch the chatbot

```bash
.venv/bin/streamlit run app.py
```

## Evaluation

RAGAS metrics with target scores ‚â• 0.8:

| Metric | Measures |
|---|---|
| Faithfulness | Answer grounded in retrieved context |
| Answer relevancy | Answer addresses the question |
| Context precision | Retrieved chunks are relevant |
| Context recall | All relevant chunks were retrieved |

**Improvement loop:**
- Low context precision ‚Üí adjust `alpha` or chunk size
- Low context recall ‚Üí increase `top_k`
- Low faithfulness ‚Üí tighten the prompt
- Low answer relevancy ‚Üí tune prompt or retrieval
